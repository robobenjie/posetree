posetree.pose

posetree is a module for representing and manipulating poses in 3D space. There are two main classes: Pose and PoseTree.

A Pose is an immutable location and orientation in 3D space, relative to a frame (i.e. 1 meter forward in X from the robot base).

A PoseTree is a object that keeps track of frames as they change over time, so that poses can know how to transform themselves into other frames, and do math operations with poses in different frames at particular points in time. You will need to subclass PoseTree to hook it up to your robot's odometry system.

241    def translate(self, translation: Sequence[float], *, frame: str = None) -> "Pose":
242        """Return a new pose translated by a vector.
243
244        The translation is relative to the basis of the pose being translated (i.e body-fixed) rather than the parent frame.
245        That means that
246        ```python
247        p2 = p1.translate((1, 0, 0))
248        ```
249        and
250        ```python
251        p2 = p1.in_frame("some_other_frame").translate((1, 0, 0)).in_frame(p1.frame)
252        ```
253        are identical, because the parent frame of a pose does not affect its translation.
254
255        Args:
256            translation: The vector to translate by.
257            frame: The frame that the vector is relative to.
258        
259        Returns:
260            A new pose translated by a vector.
261        """
262        if frame is None:
263            return Pose(Transform(self.position + self.rotation.apply(translation), self.rotation), self.frame, self.pose_tree)
264        else:
265            self_frame_t_frame = self.pose_tree.get_transform(self.frame, frame)
266            return Pose(Transform(self.position + self_frame_t_frame.rotation.apply(translation), self.rotation), self.frame, self.pose_tree)

268    def rotate_about_axis(self, axis: Sequence[float], angle: float, *, frame: Optional[str]=None) -> "Pose":
269        """Return a new pose rotated about an axis.
270
271        This is rotated in the basis of the pose being rotated (i.e body-fixed) rather than the parent frame, and it only
272        rotates the orientation. This is *not* a rotation about the origin, but a change of the orientation of the pose.
273
274        If you pass a frame, the axis will be interpreted as being expressed in that frame.
275        
276        Args:
277            axis: The axis to rotate about.
278            angle: The angle to rotate by, in radians.
279            frame: The frame that the axis is relative to. If None, the axis is interpreted in the basis vectors of the pose (not the parent frame).
280        
281        Returns:
282            A new pose with the same location but rotated about an axis.
283        """
284        axis = np.array(axis)
285        if (np.inner(axis, axis) - 1) > 1e-6:
286            raise ValueError("Axis must be a unit vector.")
287        if frame is None:
288            new_rotation = self.rotation * Rotation.from_rotvec(axis * angle)
289        else:
290            frame_t_self_frame = self.pose_tree.get_transform(frame, self.frame)
291            new_rotation = Rotation.from_rotvec(frame_t_self_frame.rotation.apply(axis) * angle) * self.rotation
292        return Pose(Transform(self.position, new_rotation), self.frame, self.pose_tree)

404    def _angle_about_axis_to(self, target: "Pose", axis: Sequence[float]) -> float:
405        """Return the signed angle between two poses' orientations about an axis. Ignores position. Will convert to the same frame.
406        
407        Args:
408            target: The target pose.
409            axis: The axis to rotate about.
410        
411        Returns:
412            The angle between two poses.
413
414        Raises:
415            ValueError: If the rotation is more than 20% out of the expected axis.
416        """
417        relative_rotation = self.transform.rotation.inv() * target.in_frame(self.frame).transform.rotation
418        relative_rotvec = relative_rotation.as_rotvec()
419        in_axis_rotation = np.dot(relative_rotvec, axis)
420        out_of_axis_rotation = np.linalg.norm(relative_rotvec - in_axis_rotation * np.array(axis))
421        if abs(in_axis_rotation) == 0 and abs(out_of_axis_rotation) > 0 or abs(out_of_axis_rotation) / abs(in_axis_rotation) > 0.2:
422            raise ValueError("Rotation between poses is more than 20% out of expected axis.")
423        return in_axis_rotation

470    def _point_at(self, target: "Pose", axis: Sequence[float], fixed_axis: str=None) -> "Pose":
471        """Return a new pose rotated to point at a target pose.
472        
473        Args:
474            target: The target pose to point at.
475            axis: The axis to rotate about.
476            fixed_axis: An optional axis ('x', 'y' or 'z') to keep fixed. If not provided,
477                the pose will be rotated the minimum rotation to have the axis point at the target pose..
478        
479        Returns:
480            A new pose with the same location but rotated to point at a target pose.
481        
482        """
483        delta = target.in_frame(self.frame).position - self.position
484        delta_local = self.rotation.inv().apply(delta)
485        delta_rotation = Rotation.align_vectors(np.array([delta_local]), np.array([axis]))[0]
486        if fixed_axis is None:
487            return self.with_rotation(self.rotation * delta_rotation)
488        if fixed_axis == 'x':
489            ry, rz, rx = delta_rotation.as_euler('yzx')
490            return self.with_rotation(self.rotation * Rotation.from_euler('xyz', [rx, 0, 0]))
491        elif fixed_axis == 'y':
492            rx, rz, ry = delta_rotation.as_euler('xzy')
493            return self.with_rotation(self.rotation * Rotation.from_euler('yxz', [ry, 0, 0]))
494        elif fixed_axis == 'z':
495            rx, ry, rz = delta_rotation.as_euler('xyz')
496            return self.with_rotation(self.rotation * Rotation.from_euler('zxy', [rz, 0, 0]))
497        else:
498            raise ValueError("fixed_axis must be one of 'x', 'y', or 'z'.")

505class PoseTree(ABC):
506    """Abstract base class to manage poses and frames.
507    
508    To hook up this library to your project, you should subclass this class and hook it up to your stream of transforms
509    by implementing _get_transform. Usually this will look like adding a callback-subscribe method to your class and storing
510    the transforms between important frames as they come in. If you don't already have a way of interpolating between transforms,
511    check out ROS's tf2 library. This library is designed to wrap nicely on top of tf2, if you are already using it.
512    
513    
514    """
515    def get_transform(self, parent_frame: str, child_frame: str, timestamp: float) -> "Transform":
516        """Return the transform from the parent frame to the child frame."""
517        return self._get_transform(parent_frame, child_frame, timestamp)
518    
519    def get_pose(self, position: Sequence[float], quaternion: Sequence[float], parent_frame: str) -> Pose:
520        """Create a pose object from a position, quaternion, and parent frame. Essentially syntactic sugar on top of Pose's constructor."""
521        return Pose(Transform.from_position_and_quaternion(position, quaternion), parent_frame, self)

523    @contextmanager
524    def temporary_frame(self, pose: Pose) -> str:
525        """Create a temporary frame from a pose.
526        
527        This is intended to be used for anonymous frames for doing calculations.
528
529        Example 1, find an approach angle that is close to the robot base:
530        ```python
531        for theta in np.linspace(0, 2 * np.pi, 100):
532            with tree.temporary_frame(target_pose.rotate_about_z(theta)) as approach_frame:
533                if abs(robot_base_pose.in_frame(approach_frame).y) < 0.1:
534                    return theta
535        ```
536
537        Example 2, find a person that is ready to talk to the robot:
538        ```python
539        for person in people:
540            with tree.temporary_frame(person.head_pose) as head_frame:
541            x, y, _ = robot_base_pose.in_frame(head_frame).position
542                if abs(y) < 0.5 and 0.5 < x < 1.5:
543                    return person
544        ```
545        """
546        name = uuid.uuid4().hex
547        self.add_frame(pose, name)
548        try:
549            yield name
550        finally:
551            self.remove_frame(name)

597class CustomFramePoseTree(PoseTree, ABC):
598    """A abstract PoseTree implementation that implements custom frames.
599    
600    This is useful if your underlying pose structure doesn't support custom frames. You subclass this
601    class and implement the _get_transform method and get full custom frame support. In a pinch you can implement
602    _get_transform to raise a NotImplemented error and use 'add frame' but it is not particularly efficient and does
603    not implement interpolation to particular timestamps, which will cause problems if you are getting sensor
604    information off a moving robot.
605    """
606    def __init__(self) -> None:
607        self.custom_frames = {}
608
609    def add_frame(self, pose: Pose, name: str) -> None:
610        """Add a frame to the tree. This is assumed to be a static frame, and will not be interpolated.
611
612        If that frame is already present, this will update the frame to a new pose.
613
614        Args:
615            pose: The pose of the new frame relative to the parent frame.
616            name: The name of the new frame.
617        """
618        self.custom_frames[name] = pose
619
620    def remove_frame(self, name: str) -> None:
621        """Remove a custom frame from the tree.
622        
623        This method is idempotent. If the frame is not present, it will do nothing.
624        """
625        if name in self.custom_frames:
626            del self.custom_frames[name]
627
628    def get_transform(self, parent_frame: str, child_frame: str, timestamp: Optional[float] = None) -> "Transform":
629        """Return the transform from the parent frame to the child frame, at the timestamp."""
630        if child_frame == parent_frame:
631            return Transform.identity()
632        if child_frame in self.custom_frames:
633            child_root_pose_child = self.custom_frames[child_frame]
634            parent_t_child_root = self.get_transform(parent_frame, child_root_pose_child.frame, timestamp)
635            return parent_t_child_root * child_root_pose_child.transform
636        elif parent_frame in self.custom_frames:
637            parent_root_pose_parent = self.custom_frames[parent_frame]
638            parent_root_t_child = self.get_transform(parent_root_pose_parent.frame, child_frame, timestamp)
639            parent_t_parent_root = parent_root_pose_parent.transform.inverse
640            return parent_t_parent_root * parent_root_t_child
641        else:
642            return self._get_transform(parent_frame, child_frame, timestamp)
643

647        """Return the transform from the parent frame to the child frame."""

650class Transform(object):
651    """A homogenous transform from a pose or frame to another location and rotation.
652    
653    In posetree, a Transform is a verb. It describes how to transform a pose from one frame to another.
654    Almost all the time, you want to turn your raw data objects (coming from forward kinematics or perception or 
655    localization) into a Pose object as soon as possible, and only go through Transforms at the borders of your sdk.
656
657    Using Transform objects instead of Pose objects is usually a code smell, unless you're doing something really
658    mathy with a bunch of intermediate transforms.
659    """
660
661    def __init__(self, position: Sequence[float], rotation: Rotation) -> None:
662        """Create a transform from a position and rotation.
663        
664        Args:
665            position: The position of the transform.
666            rotation: The rotation of the transform.
667        """
668        self._position = np.array(position)
669        self._rotation = rotation
670
671    @classmethod
672    def from_position_and_quaternion(cls, position: Sequence[float], quaternion: Sequence[float]) -> "Transform":
673        """Create a transform from a position and quaternion.

676            position: The position of the transform.
677            quaternion: The quaternion of the transform, in xyzw order.
678        
679        Returns:
680            A new transform.

692

694
695        t1 * t1.inverse == Transform.identity()

697        return Transform(-self.rotation.inv().apply(self.position), self.rotation.inv())

749        """Return the 4x4 matrix representation of the transform."""
750        matrix = np.eye(4)
751        matrix[:3, :3] = self.rotation.as_matrix()
752        matrix[:3, 3] = self.position
753        return matrix

757        """Create a transform from a matrix.
758

760            matrix: The matrix of the transform.

763            A new transform.
764        """

768    def apply(self, other: Union["Transform", Sequence[float], Sequence[Sequence[float]]]) -> "Transform":
769        """Multiply this transform by another transform or vector.
770
771        If other is a transform, this is equivalent to applying this transform, then the other transform.
772        If other is a vector, this is equivalent to applying this transform to the vector.
773        
774        Args:
775            other: The transform or vector to multiply.
776        
777        Returns:
778            The product of the two transforms.
779        """
780        if isinstance(other, Transform):
781          return Transform(self.position + self.rotation.apply(other.position), self.rotation * other.rotation)
782        else:
783            other = np.array(other)
784            return self._position + self.rotation.apply(other)
785        
786    __mul__ = apply

788    def almost_equal(self, other: "Transform", atol: float=1e-8) -> bool:
789        """Check if two transforms are almost equal.
790        
791        Handles floating point error in the position and rotation, and the fact that for quaternions negating 
792        the vector gives the same rotation.
793        """
794        # Check if positions are almost equal.
795        if not np.allclose(self._position, other._position, atol=atol):
796            return False
797        # Check if rotations are almost equal.
798        # Since quaternion [x, y, z, w] represents the same rotation as [-x, -y, -z, -w], we also need to check the inverse quaternion.
799        if not (np.allclose(self._rotation.as_quat(), other._rotation.as_quat(), atol=atol)
800                or np.allclose(self._rotation.as_quat(), -other._rotation.as_quat(), atol=atol)):
801            return False
802        return True

804    def interpolate(self, other: "Transform", alpha: float) -> "Transform":
805        """Interpolate between two transforms.
806        
807        Args:
808            other: The other transform to interpolate with.
809            alpha: The interpolation factor. 0 gives this transform, 1 gives the other transform.
810        
811        Returns:
812            The interpolated transform.
813        """
814        # Combine the rotations into a single Rotation object
815        rotations = Rotation.from_quat(np.vstack((self.rotation.as_quat(), other.rotation.as_quat())))
816        slerp = Slerp([0, 1], rotations)    
817        new_rotation= slerp([alpha])[0]
818        return Transform(self.position * (1 - alpha) + other.position * alpha, new_rotation)

820    def angle_to(self, target: "Transform") -> float:
821        """Return the angle between two transforms' orientations. Ignores position.
822        
823        Args:
824            target: The target transform.
825        
826        Returns:
827            The angle between two transform, in radians.
828        """
829        relative_rotation = self.rotation.inv() * target.rotation
830        return np.linalg.norm(relative_rotation.as_rotvec())
831    
832    def __eq__(self, other: "Transform") -> bool:
833        """Check if two transforms are equal."""
834        return self.almost_equal(other, atol=0)

838        return f"Transform(position={self.position}, rotation={self.rotation.as_quat()})"

A Pose is an immutable location and orientation in 3D space. A pose is defined with a parent_frame (where to start) and a transform (how to get to the pose from the parent frame).

Note: While a Pose is immutable, the parent frame can (and does!) change over time relative to other frames, meaning that an individual pose can move relative to other frames. (For example a pose defined in the "robot" frame will conceptually move as the robot moves relative to a 'world' frame, even though its position and orientation remain immutably constant.)

612

A homogenous transform from a pose or frame to another location and rotation.

In posetree, a Transform is a verb. It describes how to transform a pose from one frame to another. Almost all the time, you want to turn your raw data objects (coming from forward kinematics or perception or localization) into a Pose object as soon as possible, and only go through Transforms at the borders of your sdk.

Using Transform objects instead of Pose objects is usually a code smell, unless you're doing something really mathy with a bunch of intermediate transforms.
